# CAPTCHA Triggering Rules

Documentation project created as part of a technical writing task for the JetBrains Technical Writer Internship.

This repository contains an article explaining how CAPTCHA protection is triggered in the service to prevent automated abuse while maintaining a smooth user experience for legitimate users.

The documentation focuses on explaining the system behavior in a clear and structured way for technical specialists who may not be developers.

Live version: **https://nataliia-ruda.github.io/captcha-rules/**

## About

The article explains the logic behind CAPTCHA protection and answers common questions about how and why it appears.

Topics covered include:

- What CAPTCHA is and why it is used
- Why CAPTCHA should not appear on every request
- The rules used by the service to trigger CAPTCHA verification
- What happens after CAPTCHA is triggered
- How CAPTCHA helps balance security and user experience
- Common questions about CAPTCHA behavior

The documentation is written for:

- Technical support specialists  
- Product specialists  
- QA engineers  
- Other non-developer technical roles  

The goal is to explain the system behavior without requiring knowledge of the underlying codebase.

## Technologies

This project follows a Docs-as-Code workflow, where documentation is written, versioned, and deployed using development tools and CI pipelines.

The documentation was authored in IntelliJ IDEA using the JetBrains Writerside plugin.
